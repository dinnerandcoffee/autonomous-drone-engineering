# Week 12. 제어를 위한 Python 기초

## 🎯 이번 주 목표

이번 주의 목표는  
**Python을 ‘프로그래밍 언어’가 아니라 ‘제어를 위한 도구’로 사용하기 시작하는 것**이다.

자율 시스템에서 Python은:
- 센서 데이터를 다루고
- 상태를 판단하며
- 액추에이터를 제어하는

중앙 제어 언어 역할을 한다.

이번 주에는 문법 암기보다,
**제어 흐름을 표현하는 방법**에 집중한다.

---

## 🧠 핵심 개념

### 1. 제어는 흐름이다
제어 프로그램의 핵심은
값 자체가 아니라 **흐름**이다.

- 입력(Input)
- 판단(Decision)
- 출력(Output)

이 구조를 코드로 표현하는 것이 목표다.

---

### 2. 변수와 상태(State)
제어에서는 변수가 곧 상태다.

- 현재 속도
- 센서 값
- 모드(자동/수동)

상태를 명확히 정의하지 않으면
제어 로직이 복잡해진다.

---

### 3. 반복과 조건의 역할
제어 프로그램은 대부분:
- 반복(loop)
- 조건(if)

으로 구성된다.

언제 반복하고,
언제 멈출지에 대한 기준이
제어의 안정성을 결정한다.

---

## 🛠️ 실습 / 작업 내용

### 1. 간단한 제어 루프 작성
- 입력값을 받아
- 조건을 판단하고
- 출력값을 변경하는

기본 제어 루프 작성

---

### 2. 상태 변수 추가
- 상태 값을 변수로 정의
- 상태에 따라 다른 동작 수행
- 상태 전이 조건 정리

---

### 3. 로그 출력과 디버깅
- print를 활용한 상태 출력
- 흐름이 어떻게 진행되는지 확인
- 오류 발생 지점 파악

---

## 📁 산출물

- Python 제어 스크립트 (.py)
- 상태 흐름 다이어그램 또는 메모
- 실행 로그 캡처

---

## ❗ 문제와 해결

### 문제 1. 코드가 길어지면 헷갈린다
- 원인: 역할이 섞인 코드 구조
- 해결:
  입력 / 판단 / 출력 블록을 구분한다

---

### 문제 2. 프로그램이 멈추지 않는다
- 원인: 종료 조건 미정의
- 해결:
  루프 종료 조건을 명확히 작성한다
