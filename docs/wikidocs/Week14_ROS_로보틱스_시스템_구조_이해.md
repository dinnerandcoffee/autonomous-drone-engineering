# Week 14. ROS 로보틱스 시스템 구조 이해

## 🎯 이번 주 목표

이번 주의 목표는  
**ROS를 ‘어려운 프레임워크’가 아니라 ‘로봇 시스템을 연결하는 구조’로 이해하는 것**이다.

ROS는 하나의 프로그램이 아니라:
- 여러 노드(Node)가
- 메시지(Message)로 통신하며
- 시스템처럼 동작하는 구조

를 가진다.

이번 주에는 문법보다,
**ROS가 왜 이런 구조를 가지는지**에 집중한다.

---

## 🧠 핵심 개념

### 1. ROS는 운영체제가 아니다
ROS는 운영체제가 아니라,
**로봇 소프트웨어를 구성하기 위한 미들웨어**다.

- 센서 노드
- 제어 노드
- 판단 노드

이 각각이 독립적으로 실행되며,
필요할 때 메시지로 연결된다.

---

### 2. 노드(Node)와 토픽(Topic)
ROS 시스템의 기본 구성 요소는:
- 노드(Node): 하나의 실행 단위
- 토픽(Topic): 데이터 통신 채널

노드는 토픽을 통해:
- 데이터를 발행(Publish)
- 데이터를 구독(Subscribe)

한다.

---

### 3. 분산 구조의 장점
ROS의 분산 구조는:
- 모듈 교체 용이
- 디버깅 편의성
- 시스템 확장성

을 제공한다.

이 구조는 자율 시스템에 매우 적합하다.

## ❗ 문제와 해결

### 문제 1. 전체 구조가 한 번에 안 보인다
- 원인: 노드와 토픽 개념 혼동
- 해결:
  기능 단위로 노드를 나누어 생각한다

---

### 문제 2. 데이터 흐름이 헷갈린다
- 원인: 토픽 방향 인식 부족
- 해결:
  publish / subscribe 관계를 화살표로 표현한다
