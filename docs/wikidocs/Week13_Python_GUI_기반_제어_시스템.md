---

## 🧠 핵심 개념

### 1. GUI는 제어의 창구다
GUI는 단순한 화면이 아니라,
제어 시스템과 사용자 사이의 **입·출력 창구**다.

- 버튼: 명령 입력
- 슬라이더: 연속 값 조절
- 표시창: 상태 피드백

각 요소는 명확한 역할을 가져야 한다.

---

### 2. 이벤트 기반 제어
GUI 제어는 대부분 **이벤트 기반**으로 동작한다.

- 버튼 클릭
- 값 변경
- 타이머 이벤트

이벤트 발생 시
어떤 제어 로직이 실행되는지를 명확히 연결해야 한다.

---

### 3. 제어 로직과 UI 분리
안정적인 시스템을 위해서는:
- 제어 로직
- UI 코드

를 분리하는 것이 중요하다.

이 분리는:
- 유지보수성
- 확장성
- 오류 감소

에 큰 도움이 된다.

---
