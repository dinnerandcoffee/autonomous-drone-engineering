# Week 05. SOLIDWORKS 스케치와 피처 전략

## 🎯 이번 주 목표

이번 주의 목표는  
**CAD 모델링을 ‘형상 만들기’가 아니라 ‘설계 의도 표현’으로 접근하는 것**이다.

SOLIDWORKS를 사용해:
- 무작정 피처를 쌓는 방식에서 벗어나
- 스케치 단계에서 구조를 결정하고
- 수정과 확장이 쉬운 모델을 만드는 전략

을 익힌다.

이번 주를 마치면  
**나중에 바뀌어도 무너지지 않는 CAD 모델의 기초**를 갖추는 것이 목표다.

---

## 🧠 핵심 개념

### 1. 스케치는 설계 의도의 시작점
스케치는 단순한 윤곽선이 아니다.  
**설계자가 무엇을 중요하게 생각했는지 드러내는 표현**이다.

- 기준선과 기준면
- 대칭과 반복
- 치수의 우선순위

이 스케치 단계에서 결정된다.

---

### 2. 완전 정의(Full Definition)의 의미
SOLIDWORKS에서 스케치가 완전 정의되었다는 것은:
- 모든 요소가 의도대로 고정되어 있고
- 불필요한 자유도가 남아 있지 않다는 뜻이다.

완전 정의된 스케치는:
- 예측 가능하고
- 수정에 강하며
- 오류를 줄인다.

---

### 3. 피처 순서와 설계 안정성
같은 형상이라도
피처를 만드는 순서에 따라 모델의 안정성이 달라진다.

- 기준 형상 먼저
- 반복 요소는 나중에
- 변경 가능성이 큰 요소는 뒤쪽에

이 원칙이 중요하다.

---

## 🛠️ 실습 / 작업 내용

### 1. 기준 스케치 만들기
- 기준면(Front/Top/Right) 중 하나 선택
- 전체 형상을 대표하는 스케치 작성
- 대칭과 기준선을 적극 활용

---

### 2. 기본 피처 생성
- Extrude, Cut 같은 기본 피처 사용
- 불필요한 디테일은 넣지 않는다
- 형상의 ‘뼈대’를 먼저 만든다

---

### 3. 수정 실험
- 초기 스케치 치수 변경
- 모델 전체가 어떻게 변하는지 관찰
- 깨지는 피처가 있다면 원인 분석

---

## 📁 산출물

- SOLIDWORKS 파트 파일 (.SLDPRT)
- 기준 스케치가 명확한 모델
- 치수 변경 테스트 결과

---

## ❗ 문제와 해결

### 문제 1. 수정하면 모델이 깨진다
- 원인: 스케치/피처 의존 관계 불명확
- 해결:
  기준 스케치 → 주요 피처 → 세부 피처
  순서를 다시 정리한다

---

### 문제 2. 스케치가 복잡해진다
- 원인: 한 스케치에 모든 정보를 담으려 함
- 해결:
  기능 단위로 스케치를 나눈다
