# Week 20. 드론 DIY 제작 및 IoT 연동

## 🎯 이번 주 목표

이번 주의 목표는  
**설계된 드론을 실제로 제작하고, 외부 시스템과 연결하여 운용 가능한 상태로 만드는 것**이다.

이 단계에서는:
- CAD와 현실의 차이
- 조립 과정에서의 변수
- 운용 환경과의 연결

을 직접 경험하게 된다.

이번 주를 마치면  
**설계 → 제작 → 운용으로 이어지는 전 과정을 한 번 완주**하게 된다.

---

## 🧠 핵심 개념

### 1. 제작은 설계의 검증이다
제작 과정은
설계를 평가하는 가장 솔직한 시험이다.

- 조립이 되는가?
- 공차는 적절한가?
- 예상하지 못한 간섭은 없는가?

제작 중 발생하는 문제는
모두 설계 피드백이다.

---

### 2. DIY 조립의 우선순위
모든 것을 한 번에 완벽히 만들 필요는 없다.

권장 우선순위:
1. 구조적 안정성
2. 전원 연결 안정성
3. 기본 제어 동작
4. 추가 기능

안전이 항상 최우선이다.

---

### 3. IoT 연동의 역할
IoT 연동은 드론을:
- 원격 모니터링 가능하게 하고
- 데이터 수집 대상으로 만들며
- 시스템 확장의 출발점이 된다.

드론은 단독 장치가 아니라
네트워크의 일부가 된다.

---

## 🛠️ 실습 / 작업 내용

### 1. 실제 제작 및 조립
- 프레임 출력 또는 가공
- 모터, ESC, 컨트롤러 장착
- 배선 및 고정

---

### 2. 기본 동작 테스트
- 전원 인가 테스트
- 모터 회전 확인
- 센서 인식 확인

(프로펠러는 분리한 상태로 진행)

---

### 3. IoT 연동 설정
- Wi-Fi 또는 통신 모듈 설정
- 상태 데이터 송신
- 외부 대시보드 또는 로그 확인

---

## 📁 산출물

- 조립 완료 드론 사진
- 배선 및 내부 구조 정리 문서
- IoT 연동 로그 또는 화면 캡처

---

## ❗ 문제와 해결

### 문제 1. 조립이 잘 맞지 않는다
- 원인: 공차 또는 출력 오차
- 해결:
  설계를 수정하거나 조립 순서를 변경한다

---

### 문제 2. 통신이 불안정하다
- 원인: 전원 노이즈 또는 신호 간섭
- 해결:
  전원 분리 또는 배선 재정리
